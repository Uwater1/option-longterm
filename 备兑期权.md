## 1. 这个策略是做什么的？

### 1.1 一句话总结

**持有ETF现货，卖出虚值认购期权赚取权利金。当过滤器提示市场过热时，完全放弃卖Call，保留现货弹性，规避大跌或暴涨带来的行权损失。** 这就是本策略的核心。

> [!NOTE]
> 经过回测审计与实证优化，策略已**移除保护性看跌期权（Long Put）**，以消除长期买Put带来的高额负拖累。同时，废除了过热时的 "OTM4档（腿C）强制卖出" 机制，改为**在过热周期不卖Call（Skip Call Leg C）**，这极大避免了在市场暴涨行情中被低价行权造成的踏空损失。

### 1.2 为什么要做这个策略？

A股市场有一个特点：**震荡和下跌的时间远多于单边上涨的时间。** 

根据Alpha Finder对沪深300指数（2010年至今）的30日收益率分布统计：
- **30日内下跌超过0.05元（约1个单位）的概率：36.26%**
- **30日内上涨超过0.05元的概率：42.33%**
- 大部分时间市场在小幅波动，这恰好是期权卖方（卖Call）最容易赚钱的环境

**核心逻辑：** 既然大部分时间市场不会暴涨，那卖出行权价高于当前价的Call（认购期权），大概率不会被行权，权利金就是纯利润。

### 1.3 策略的双腿操作

本策略每次交易主要涉及2个Call腿（Leg），移除了原有的Put保护腿以最大化净收益：

| 腿 | 操作 | 方向 | 作用 | 举例 |
|:---|:---|:---|:---|:---|
| **腿A：备兑Call** | 卖出1张OTM2档认购 | 收钱 | 主要收入来源 | ETF价格4.80，卖出K=4.90的Call |
| **腿B：远端Call** | 卖出1张OTM3档认购 | 收钱 | 补充收入，保留上涨弹性 | ETF价格4.80，卖出K=5.00的Call |

> 当技术指标提示市场过热时，策略会**直接跳过开仓（Skip OTM4）**，不建立任何认购期权空头，保留ETF多头资产的完整上涨空间。

## 3. 策略具体怎么执行？

### 3.1 每月操作流程

```
月初（上月合约到期后第一个交易日）
  │
  ├── 第1步：检查技术指标（不同品种采用特定优化过滤器组合）
  │     │
  │     ├── ✅ 过滤器通过（价格温和，未超买/大涨）
  │     │     → "可以收租"，积极备阅开仓
  │     │     → 卖出 2张Call（OTM2档 + OTM3档）
  │     │
  │     └── ❌ 过滤器未通过（市场过热，超买或突破）
  │           → "空仓避险"，本月不做任何期权操作 (Skip OTM4)
  │           → 保留底层ETF资产的完整上涨权利
  │
  └── 第2步：持有到期（不提前平仓，无需买Put对冲）
        │
        ├── Call未被行权 → 权利金全部保留为净利润 ✅
        └── Call被行权 → ETF被划拨（踏空部分，转入现金做下一轮CSP操作）
```

### 3.2 持仓规模

| 项目 | 数值 | 说明 |
|:---|:---|:---|
| ETF持仓 | 20,000份 | 作为2张Call的备兑抵押 |
| Call合约 | 0 ~ 2张 | 市场未过热时卖出，每张对应10,000份ETF |
| Put合约 | 0张 | 已废除 |
| 乘数 | 10,000 | A股ETF期权标准乘数 |

### 3.3 成本模型

| 费用项 | 数值 | 说明 |
|:---|:---|:---|
| 买卖价差 | ±2% | 卖出按mid×0.98，买入按mid×1.02 |
| 手续费 | 2元/腿 | 每张合约每笔2元 |
| 行权费 | 0.6元 | 仅当作为买方行权时 |

## 4. 关键技术细节

### 4.1 IV Rank（隐含波动率排名）

IV Rank衡量当前隐含波动率在过去252天中的排名位置：

```
IVR = (当前IV - 252天最低IV) / (252天最高IV - 252天最低IV)
```

- IVR高（>0.50）→ 市场波动大，卖更远虚值的Call
- IVR低（<0.10）→ 市场平稳，可以卖近一点的Call
- 当前300ETF的IVR约为0.06，属于低波动环境

### 4.2 RSI（相对强弱指标）

- 使用14日RSI
- RSI > 66 → 市场短期可能过热，减少Call仓位
- 50ETF的阈值更低（RSI > 60即视为过热）

### 4.3 布林带（Bollinger Bands）

- 使用20日均线 ± 2倍标准差
- 收盘价突破上轨 → 短期过热信号
- 与RSI联合使用，两个条件同时满足才算"通过"过滤器

### 4.4 过滤器效果对比

| 场景 | 卖Call OTM2 胜率 | 卖Call OTM2 期望收益 |
|:---|:---|:---|
| 过滤器通过（RSI<66 且 价格<BB上轨） | **97%** | **+1,726元** |
| 被过滤掉（RSI≥66 或 价格≥BB上轨） | 56% | +712元 |

过滤器把最危险的9个周期（约13%）剔除出去，这些周期胜率只有55%左右。保留的周期胜率高达97%。

## 6. 过滤器研究框架

过滤器用于在开仓前判断市场状态，避免在过热环境下卖出Call。当前使用的技术指标过滤器包括：

### 6.1 已实现的过滤器

| 编号 | 指标 | 条件 | 含义 |
|:-----|:-----|:-----|:-----|
| f1 | SMA20 | Close < SMA20 | 价格低于20日均线 |
| f2 | EMA20 | Close < EMA20 | 价格低于20日指数均线 |
| f3 | RSI14 | RSI < 70 | 未严重超买 |
| f4 | RSI14 | RSI > 30 | 未严重超卖 |
| f5 | MACD | MACD Histogram < 0 | 空头动能 |
| f6 | 布林带 | Close < Upper BB(20,2) | 价格未突破上轨 |
| f9 | ROC20 | ROC < 5% | 20日涨幅不大 |
| f10 | ATR突破 | Close < SMA20 + ATR(20) | 未突破ATR通道 |
| f11 | ADX14 | ADX < 25 | 趋势不强烈 |
| f12 | EMA交叉 | EMA10 < EMA30 | 短期均线在长期下方 |
| f15 | 随机指标 | %K < 80 | 未超买 |
| f16 | 20日高点 | Close < 0.98 × 20日最高价 | 未接近近期高点 |
| f17 | 252日高点 | Close < 0.95 × 252日最高价 | 未接近年度高点 |
| f20 | Keltner | Close < EMA20 + ATR(20) | 未突破Keltner通道 |

### 6.2 过滤器评估工具

**单过滤器评估**（`eval_synth_filters.py`）：
- 在合成期权数据上逐个测试每个过滤器
- 输出每个过滤器的放置率（Placement Rate）、期望收益、最大亏损
- 用于初步筛选有效指标

**组合过滤器搜索**（`eval_synth_combinations.py`）：
- 测试27种过滤器组合（AND/OR逻辑）
- 对组合进行评分：`score = PR × ER_passed + MaxLoss/100 - ER_filtered`
- 筛选条件：放置率 ≥ 70%，过滤掉的周期期望亏损 < 1000元
- 输出排名前5的组合到 `filter_syntetic.txt`

**真实数据验证**（`evaluate_combinations.py`）：
- 在真实期权数据上验证合成数据的结论
- 使用 `research_otm_levels.py` 的回测框架
- 输出排名前5的详细报告到 `filter2_*.log`

### 6.3 研究流程

```
1. eval_synth_filters.py    → 找出有效的单过滤器
2. eval_synth_combinations.py → 搜索最佳组合
3. evaluate_combinations.py   → 真实数据验证
4. backtest_covered_call.py   → 最终回测确认
```

## 7. 风险管理

| 风险场景 | 发生概率 | 影响 | 应对措施 |
|:---|:---|:---|:---|
| **市场温和上涨** | 高 | Call被行权，ETF被卖出 | 转入"现金担保卖Put"模式，等待回调接回底仓 |
| **市场暴跌** | 低 | ETF浮亏 | 已废除买Put对冲，依靠震荡市的高胜率卖Call收益来长期覆盖风险 |
| **市场横盘** | 最高 | 最佳场景 | 权利金全部保留为纯利润 |
| **市场暴涨** | 极低 | Call被行权，错失上涨收益 | 远端Call腿保留部分上涨弹性 |

### 7.1 最大回撤

根据300ETF回测，策略最大回撤出现在2026年3~4月（中美贸易摩擦升级导致市场剧烈波动），单周期最大亏损约3,385元。但整体回撤恢复较快，因为后续月份的权利金收入会逐步弥补。

### 7.2 被行权后的处理

如果Call被行权（ETF被卖掉）：
1. 不需要恐慌——你已经赚了权利金
2. 立即转为"现金担保卖Put"（Cash-Secured Put）
3. 卖出平值或轻度虚值的Put，等待市场回调
4. 如果Put被行权，你就以更低的价格买回ETF
5. 如果Put未被行权，你赚了权利金，下月再卖一次

## 8. 架构设计

### 8.1 数据流

```
rqdatac → update_data.py → data/*.parquet → 所有分析脚本
```

- rqdatac 是 RiceQuant 的数据接口，安装在系统 miniconda 中
- `update_data.py` 负责全量/增量下载，并清除旧 IV 缓存
- 所有下游脚本从 Parquet 文件读取数据，不直接连接数据源

### 8.2 回测引擎（backtest_covered_call.py）

- **周期划分：** 按月度期权到期日划分交易周期，上月到期后开仓
- **IV Rank（252天）：** 驱动 OTM 偏移量，高 IVR → 更远虚值
- **过滤器逻辑：** 根据不同ETF动态选择：
  - 300ETF: `RSI < 66 且 Close < BBU`
  - 50ETF: `RSI < 70 且 Close < SMA20 + ATR20`
  - 500ETF: `(RSI < 70 或 10日ROC < 5%) 且 Close < BBU`
- **跳过OTM4（Skip OTM4）：** 过滤器不通过时，完全不进行期权开仓（Skip OTM4），不卖 any Call，保留底层ETF的完整上涨弹性。
- **保护性 Put：** 默认关闭（NO_PUT_MODE=True），不再买入Put，消除保险费拖累。
- **限价单模式：**
  - `--model-offset`：卖 Call 使用 BS 映射模型限价单（5分钟 K 线模拟成交）
  - `--limit-entry`：买 Put 使用 BS 映射模型限价单
  - 需要预先训练 `predict_open_high.py` 模型
- **交易成本：** 买卖价差 ±2%，手续费 2元/腿

### 8.3 价差模型（spread.py）

LightGBM 模型预测 `log(1+spread)`，特征包括：中间价、IV、OTM深度、DTE、moneyness。

### 8.4 合成期权（numba_utils.py）

通过 `process_synthetic_strikes_loop()` 在两个到期日之间插值 IV，生成固定期限的合成合约。Numba JIT 编译加速 Black-Scholes 定价和 IV 求解。

### 8.5 关键参数

| 参数 | 值 | 说明 |
|:-----|:---|:-----|
| SPREAD_HALF | 0.02 | ±2% 滑点 |
| COMMISSION | 2.0 RMB | 每腿手续费 |
| ETF_SHARES | 20,000 | ETF持仓规模 |

## 9. 限价单预测模型

### 9.1 问题描述

卖出备兑 Call 时，传统做法是按 `中间价 × 0.98` 挂单（固定 2% 滑点）。实际上，如果预测当天 ETF 的最高价能涨到某个位置，就可以挂一个更紧的限价单，仍然有 90% 的概率成交。

`predict_open_high.py` 预测 ETF 当天 `(最高价 - 开盘价) / 开盘价` 的 **第10百分位数 (P10)**。如果 ETF 有 90% 概率从开盘价上涨至少 P10%，那么挂单在 `开盘价 × (1 + P10%)` 就有 90% 的成交概率。

### 9.2 模型架构

1. **特征工程**：25个候选特征，从 ETF 日线数据提取：
   - 隔夜跳空、RSI(14)、20日已实现波动率、ATR(14)
   - Close/SMA50 比值、MACD 柱状图、ROC(5/10/20)
   - 布林带宽度、成交量比值、星期几
   - MA偏离度：`(开盘 - EMA5) / EMA5`、`(开盘 - EMA20) / EMA20`
   - 前一日特征：日内振幅、开盘到最高价、隔夜跳空从昨日最高、上下影线
   - 动量震荡指标：Stochastic %K(14)、Williams %R(14)、ADX(14)、MFI(14)、CCI(20)
   - 尾部风险：20日收益率偏度

2. **特征选择**：前向逸步选择，5折时间序列交叉验证，最小化 Pinball 损失（分位数损失，q=0.10）。最多选择 6 个特征。

3. **模型训练**：
   - **Statsmodels 分位数回归**（线性，可解释系数）
   - **LightGBM 分位数回归**（非线性，特征交互）
   - 交叉验证损失差异 < 5% 时采用集成平均，否则选胜者

4. **自适应分位数搜索**：二分查找 q'（通常 0.03–0.04）使得原生覆盖率达 90%，替代固定的 q=0.10。

5. **集成袋装 (Ensemble Bagging)**：训练 5 个 LightGBM 模型（不同 bootstrap 重采样），预测时取平均，降低方差和过拟合。

6. **波动率区间校准**：按中位数 vol20 分成低波动和高波动两个区间，分别计算校准偏移量。预测时根据当前 vol20 应用对应区间的校准。

7. **Block-Bootstrap 数据增强**：20天循环块，1倍比例合成数据，进一步减少过拟合。

8. **跨 ETF 池化训练** (`--pool`)：可选模式，将 3 只 ETF 数据合并（~7200 样本）训练更稳健的模型。每只 ETF 仍单独校准。

### 9.3 回测集成

回测引擎通过以下方式集成预测模型：
- **卖 Call 限价单** (`--model-offset`)：使用 BS 映射将 ETF P10 预测映射为期权限价，用 5分钟 K 线模拟成交
- **买 Put 限价单** (`--limit-entry`)：同样使用 BS 映射预测保护性 Put 的限价买入价
- 未成交的腿按最后 5分钟收盘价强制成交

## 11. 策略的局限与改进方向

### 11.1 当前局限

- **上行空间有限：** 暴涨时收益封顶（被行权），不能享受全部涨幅
- **无Put大跌裸露：** 移除了Put保险后，在大跌月份没有期权对冲保护，多头资产价值完全随现货波动。
- **500ETF表现不佳：** 中小盘波动大，但经过 (RSI<70 或 ROC10<5%) 且 Close<BBU 过滤器优化后已实现扭亏为盈（+12.2K）。

### 11.2 后续研究方向

- [ ] 多品种横向对比：优化500ETF和1000ETF的参数
- [ ] 动态Delta对冲：极端行情下用现金调整Delta暴露
- [ ] IV Skew挖掘：利用波动率偏度优化Put的行权价选择
- [ ] 分红调整自动化：防止ETF分红除权导致回测误差
- [ ] 日内择时：研究是否有更好的入场时机（非月初开盘直接入场）
