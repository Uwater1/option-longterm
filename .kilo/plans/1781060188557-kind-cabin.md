# 500ETF Performance Improvement Research Plan

## Problem Diagnosis

500ETF underperforms due to 3 structural issues:

1. **Highest volatility (26.8% ann)** — ~40% higher than 300ETF, yet uses identical RSI threshold (66) and same OTM level selection (OTM2/OTM3 pass, OTM4 fail). Strikes too close → more assignments.
2. **Expensive put hedge** — Level 1 Long Put consistently costs 500-1000+ RMB, often exceeding call premium income. Net drag on most cycles.
3. **No 500ETF-specific research** — All filter combination searches (`eval_synth_combinations.py`, `eval_synth_filters.py`, `evaluate_combinations.py`) hardcoded to 300ETF. No synthetic data generated for 500ETF.

## Research Steps

### Step 1: Generate synthetic 500ETF data
- Run `research_synthetic_otm.py -e 500` to generate `synthetic_options_500ETF.parquet`
- Also run `research_synthetic_no_filter.py -e 500` for baseline

### Step 2: OTM level analysis on real 500ETF data
- Run `research_otm_levels.py -e 500` (with RSI<66 + BB filter)
- Run `research_otm_no_filter.py -e 500` (baseline)
- Compare OTM1-OTM5 win rates and expected returns vs 300ETF
- **Key question**: Should 500ETF use OTM3+OTM4 (pass) and OTM5/OTM6 (fail) instead?

### Step 3: Adapt filter combination research to 500ETF
- Modify `eval_synth_combinations.py` to accept ETF selector (`-e 500`)
- Run all filter combinations on synthetic 500ETF data
- Compare top combos with 300ETF results

### Step 4: Individual filter evaluation on 500ETF
- Modify `eval_synth_filters.py` to accept ETF selector
- Run all 20 individual filters on synthetic 500ETF data
- Identify which filters work for 500ETF's higher volatility regime

### Step 5: Analyze optimal parameters for 500ETF
Based on research results, determine:
- **OTM level**: How many levels further out does 500ETF need? (hypothesis: +1 or +2 levels)
- **RSI threshold**: Should 500ETF use tighter threshold (e.g., 60 like 50ETF)?
- **Put hedge level**: Test Level 2 put (cheaper) vs Level 1 — does reduced cost improve net P&L?
- **Filter combo**: Which combination works best for 500ETF specifically?

### Step 6: Implement and validate improvements
- Update `backtest_covered_call.py` with 500ETF-specific parameters
- Run backtest and compare with current 21,447 RMB baseline (45 cycles)
- Document findings

## Key Hypotheses to Test

| Hypothesis | Rationale | Test |
|------------|-----------|------|
| 500ETF needs +1-2 OTM levels | Higher vol → wider moves → need more buffer | OTM level analysis |
| RSI threshold should be ~60 (not 66) | 500ETF more volatile, overbought more dangerous | Filter eval on 500ETF |
| Put at Level 2 instead of Level 1 | Level 1 too expensive, net drag | Backtest with Level 2 put |
| Different filter combo needed | 300ETF filters may not suit 500ETF vol profile | Filter combo eval on 500ETF |
| IVR should actually be used | High IVR → further OTM was designed but unused | IVR-based OTM selection |

## Execution Priority

1. Steps 1-2 first (data generation + OTM analysis) — fastest to run, highest insight value
2. Steps 3-4 (filter research) — requires script modifications
3. Step 5 (parameter synthesis) — depends on results from 1-4
4. Step 6 (implementation) — final changes based on research
